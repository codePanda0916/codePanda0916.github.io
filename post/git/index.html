git使用小结

一、git配置说明

忽略某些文件时，需要编写.gitignore文件。 –global参数为全局参数，也就数带这个参数的命令在这台电脑的所有Git仓库下都有用。 配置Git的时候，加上–global是针对当前用户起作用的，如果不加，那只针对当前的仓库起作用。

通过全局命令进行如下操作来配置别名（避免原单词长写错，偷懒）

git config --global alias.st status
git config --global alias.ci commit
git config --global alias.co checkout
git config --global alias.br branch

以上配置好后就可以进行如下简写操作啦

完整语法 别名简写 备注
git status git st status简写为st
git commit -m ‘xxxx’ git ci -m ‘xxx’ commit简写为ci
git checkout master git co master checkout简写为co
git branch dev git br dev branch简写为br

用户名密码配置

git config --global user.name "your name"
git config --global user.email "your@example.com"

二，工作区与暂存区

第一步：用git add把文件添加进去，实际上就是把文件修改添加到.git中的暂存区。

第二步：用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。

我们初始化创建.git版本库时，git自动为我们创建了唯一一个master分支，所以，git commit就是往master分支上提交更改。

远程仓库

1，关联一个远程库

git remote add origin git@xyjxxxxx.git

2，从远程库克隆

git push -u origin master // 第一次推送，远程库是空的，加上-u参数。
git push origin master    // 第二次之后就可以去掉-u参数。

四，分支管理

1，创建分支

git branch daily/0.0.1   // 创建分支

2，切换分支

git checkout daily/0.0.1   // 切换分支
or
git switch daily/0.0.1     // 切换分支

3，创建分支+切换分支

git branch daily/0.0.1   // 创建分支
git checkout daily/0.0.1 // 切换分支
// 以上两步可以合并为如下操作
git checkout -b daily/0.0.1  // 创建分支并切换到此分支
// 新版git提供新的git switch来切换分支
git switch -c daily/0.0.1    // 新版git

4，查看当前分支

git branch // 查看当前分支，当前分支前面带个*号，此命令会列出所有分支。

5，合并分支

git merge dev //合并指定分支dev到当前分支上。
git merge --no-ff -m 'merge with' dev  // --no-ff参数表示禁用Fast forward

合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。

6，删除分支

git branch -d dev // 删除dev分支，（如果dev分支已经合并到master上，则可以将此分支删除掉）

7，隐藏当前工作状态

git stash

如果你当前工作dev分支进行到一半，还没法提交，发现线上有一个bug，急需1h内修改掉。就先用git stash将当前工作隐藏“储藏”起来，等创建另外分支修改好bug后再恢复 那么现在就在master上创建分支来修复bug

git stash                  // 第一步，影藏当前工作状态
git checkout master        // 第二步，切换到master分支
git checkout -b issDev     // 第三步，创建issDev分支+切换到issDev分支,并修改分支
git checkout master        // 第四步，修改完了，切换到master之后，开始下一步合并分支
git merge --no-off -m 'iss' issDev // 第五步，将issDev分支合并到master上
git branch -d issDec       // 第六步，删掉合并后的分支

接下来，切换到dev分支,并恢复隐藏的原工作现场

git checkout dev         // 切换到dev分支
git stash list           // 查看之前的工作现场去哪儿里，接下来恢复现场
// 提供两种方法
// 方法一：
git stash apply          // 恢复，但stash内容不会删除
git stash drop           // 删除stash内容
// 另外：如果多次git stash后，恢复加指定stash，如下
git stash apply stash@{0}
// 方法二：
git stash pop            // 恢复+同时删除stash内容

删除掉stash内容后，用git stash list就查不到任何stash内容啦

8，如何将master分支上修复的bug，合并到当前的dev分支上

git checkout dev             // 切换到dev分支上
git cherry-pick <commit_id>  // 把bug提交的修改"复制"到当前分支，避免重复劳动

9，强行删除一个分支

git branch -D testDev  // 强行删除testDev分支

10，推送分支

git push origin master   // 推送到远程的master分支上
git push origin dev      // 推送到远程的dev分支上

11，查看远程的信息

git remote       // 查看远程库的信息
git remote -v    // 查看更详细的信息

12，在本地创建和远程分支对应的分支

// 本地与远程分支的名称最好一致
git checkout -b 本地branch-name origin 远程branch-name

13，关联本地分支与远程分支

// 本地与远程分支的名称最好一致
git branch --set-upstream 本地branch-name origin 远程branch-name

14，删除关联的远程库

git remote rm origin

15，git rebase

git rebase

rebase操作可以把本地未push的分叉提交历史整理成直线； rebase的目的是使得我们在查看历史提交的变化时更容易，因为分叉的提交需要三方对比。

五，标签管理

Git的标签是版本库的快照，有标签就可以对应找到commit。

1，创建标签

git checkout dev             // 切换到dev分支上
git tag v1.0                 // 给dev打标签为v1.0
git tag                      // 查看所有标签
git tag v2.0 f232c634        // 给commit_id为f232c634的提交打标签
git show v1.0                // 查看v1.0标签信息
git tag -a v3.0 -m 'addTag'  // 创建带有说明的标签，-a指定标签名，-m指定说明文字

2，操作标签

// 删除标签(比如标签打错)
git tag -d v1.0
// 将某个标签一次性推送到远程
git push origin v1.0
// 一次性推送全部尚未推送到远程的本地标签
git push origin --tags
// 删除远程的标签（两步）
git tag -d v2.0                    // 先删除本地标签
git push origin :refs/tags/v2.0    // 再删除远程标签
